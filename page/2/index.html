分治法求最大子列和

  1. 问题分析

    对于一个序列，我们可以把它分成两个子序列，故我们得到三个字列和，分别是

    • 序列的左半部分的最大子序列和
    • 序列的右半部分的最大子序列和
    • 横跨序列左半部分和右半部分得到的最大子序列和：即对包含左半部分的最后一个元素的最大子序列和以及包含右半部分第一个元素的最大子序列和二者求和所得到的值

    故这三种情况中最大的字列和，即为最大子列和。我们可以把问题不断的划分成一个个相同的子问题，再进行合并，最终得到的字列和即为最大子列和。

贪心法求解数列极差问题

  1. 问题描述

    给定含有n个正整数的数列a，做如下两步操作：

    (1)每一次删除其中的两个数ai和aj;

    (2)在数列中加入一个数ai×aj+1

    循环执行步骤(1)(2)直到集合中只剩下一个元素为止。设计算法求得数列中剩余的数为最大值max和最小值min，则该数列的极差为M=max-min。

  2. 问题分析

    当一个数列中含有n(n>2)数时，该数列按数列极差法

    求最大数值的方法为：每次选择数列中最小的两个数进行相乘加1。

    求最小数值为：每次从数列中选择两个最大的数相乘加1。

    证明见附页1。

  3. 计算模型

    存储：数组num[n]

    计算：

    • num[n]中前两个最大值（最小值）t1，t2
    • 两个数的极大值（极小值） t1*t2+1

  4. 算法设计与描述

    输入： n个数的数组num[n]

    输出：极大值或极小值

  5. 算法分析

    问题输入规模为n，核心在于while循环的执行次数以及最大值最小值的求解。

    其中寻找最大值最小值时间复杂度为O(logn)+O(log(n-1))=O(logn)

    该函数共被执行2*(n-2)次，此外还有综合函数

    故T(n)=2*(n-2)*O(logn)+c=O(nlogn)

对象的消亡

  1. 为什么cat3没有析构函数在程序退出时未被调用：

    new出来的在堆上
    直接定义的在栈上

    栈，就是那些由编译器在需要的时候分配，在不需要的时候自动清除的变量的存储区。里面的变量通常是局部变量、函数参数等。
    堆，就是那些由new分配的内存块，他们的释放编译器不去管，由我们的应用程序去控制，一般一个new就要对应一个delete。如果程序员没有释放掉，那么在程序结束后，操作系统会自动回收。

  2. 为什么注释掉的语句不可以

    error: taking address of temporary [-fpermissive]
    A “taking address of temporary array” error means that you were trying to take the address of a temporary object. That won’t do what you think, because temporary objects self-destruct immediately after first use, so any address would be invalid before you could use it. So don’t do that. The error message should flag the code line (and even character number on that line) where the problem is. That should show you where you’re taking the address of a temporary. If you need to refer back to the contents of a temporary object, save it in a separate variable first. You can’t “re-visit” temporaries.
