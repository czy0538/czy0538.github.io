我的方法

非常幼稚直接的做法，但空间复杂度还可以。

  • 范围问题

    十分不动脑子的办法，32位int翻转后的最大数也不会超过long long的表示范围，所以输出成long long进行判断。

  • 负号问题

    stringstream处理得到的是一个绝对值，如果原数是负数乘-1。

  • x=0问题

    直接返回0

大佬做法

y == INT_{MIN, MAX}/10时可不可能溢出呢？
设参数x的最高位为a，若y == INT_{MIN, MAX}/10，则y == ±214748364a, x == ±a463847412。
x为int型的输入参数，所以a只能等于1；故 y == INT_{MIN, MAX}/10 时不可能溢出

Day15 70. 爬楼梯

这个题有很多做法，最数学的那种是直接搞成了斐波那契数列，想不到的。比较计算的还有什么动态规划之类的，也不会。这种是高中的做法。

我们可以把这个问题等效的看做对n进行分解，分解成:

$$
n=\sum_{i=0}^x{2}+\sum_{j=0}^{n-2x}{1}
$$

其中$$x=\lfloor n/2 \rfloor $$,即可以最多有多少个组成。这个问题转化成了$y=n-2*i+1$个箱子中放入完全相同的i个小球，一共有多少种放法。有排列组合相关的只是我们知道共有$C_{i+y-1}^{y-1}$，因此累加求和即可，这种算法的时间和空间耗费都是双百，但需要注意求组合数的方法，C方法只能求到41，但C2可以完成测试。

Day2 160相交链表

2020年9月20日08:58:51

难点在于本题要求时间复杂度为O(n)，空间复杂度为O(1)，而直觉的方法依次比较的方法时间复杂度为O(n**2)。我们注意到如果链表能够相交，那么从最后一个元素开始，到第一个相交的元素之间的元素一定相同，如果是数组的话，我们就可以从后向前比较找到第一个不同的元素即可。但这不是数组，但同时注意到只要从后向前找到第一个相同的指针，那么后面的指针也一定是相同的。因此我们需要找到两个链表中长的，计算出其余短的的差值，即舍弃掉长的链表前面长的部分（那部分一定不可能相交），然后依次比较，找到第一个相同的指针即可。

Day3 242. 有效的字母异位词

该题有两种思路，一种是先排序，然后再逐个比较。排序的时间复杂度O(nlogn)，比较时间复杂度O(n)，总的时间复杂度O(nlogn)，空间复杂度O(1)

第二种办法是建立一个字母表，扫描一遍第一个串，计算出每个字母的个数，然后再扫描第二个串，如果两个串字母个数不等，说明不相同，否则相同。两遍扫描时间复杂度O(n)，空间复杂度建立字母表O(1)

我们用数组result存储最终的答案。

首先，我们将列表中的区间按照左端点升序排序。然后我们将第一个区间加入 result数组中，并按顺序依次考虑之后的每个区间：

如果当前区间的左端点在数组result最后一个区间的右端点之后，那么它们不会重合，我们可以直接将这个区间加入数组 result 的末尾；

否则，它们重合，我们需要用当前区间的右端点更新数组 merged 中最后一个区间的右端点，将其置为二者的较大值。

时间复杂度：O(n\log n)O(nlogn)，其中 nn 为区间的数量。除去排序的开销，我们只需要一次线性扫描，所以主要的时间开销是排序的 O(n\log n)O(nlogn)。

空间复杂度：O(\log n)O(logn)，其中 nn 为区间的数量。这里计算的是存储答案之外，使用的额外空间。O(\log n)O(logn) 即为排序所需要的空间复杂度。

Day5 75. 颜色分类

法1：计数排序

一个直观的解决方案是使用计数排序的两趟扫描算法。
首先，迭代计算出0、1 和 2 元素的个数，然后按照0、1、2的排序，重写当前数组。

时间复杂度：扫描数组O(n),重写数组O(n)

空间复杂度：需要一个存放每种颜色的个数O(1)

法2：单指针法

我们可以考虑对数组进行两次遍历。在第一次遍历中，我们将数组中所有的 0 交换到数组的头部。在第二次遍历中，我们将数组中所有的 1 交换到头部的 0 之后。此时，所有的 2 都出现在数组的尾部，这样我们就完成了排序。开始时，头部大小为0。

这里我们有个疑问，为什么能够成功的完成交换呢？

我们用ptr指向头部的尾后元素，开始时ptr=0；假设共有n个元素，指针i初始=0，数组nums存储元素；

有两种情况：

①若nums[i]==0 ptr和i交换，同时ptr++，i++

②若nums[i]!=0,仅i++

实际上遇到元素有四种组合：

  • nums[i]==0 且 nums[ptr]!=0,即情况①

  • nums[i]！=0 且 nums[ptr] ==0，即情况②

  • nums[i]！=0 且 nums[ptr]!=0，即情况②

  • nums[i]==0 且 nums[ptr] ==0，即情况①，这种情况下两个元素交换后，两个指针均会前进，但这里有个疑问，由于i继续向下了，其实不会访问到这个0了，会不会这个0不能被加入头部呢？

Day 1 925. 长按键入

我的想法：

使用双指针，第一个指针i指向named数组，第二个指针j指向typed数组。

如果named[i]与named[i+1]相等，则count_n++，i++；若不等则当前的count_n即为named[i]对应字符的在named中的个数。此时再判断typed[j]与typed[j+1],若相等，则count_t++，j++；否则比较count_t和count_n大小关系，如果count_t大于等于 count_n,说明正确，继续指向下个字符，反之说明少输入了。依次比较，直至named或typed遍历完成。如果i不为named.size()，说明typed缺少了元素。反之说明typed正确。

其实这个思路，如果去掉一个条件，typed中可能有不包含在named中的元素也是正确的。

官方做法

根据题意能够分析得到：字符串 $$\textit{typed}$$的每个字符，有且只有两种「用途」：

作为 $$\textit{name} $$的一部分。此时会「匹配」$$\textit{name}$$中的一个字符

作为长按键入的一部分。此时它应当与前一个字符相同。

如果 $$\textit{typed}$$ 中存在一个字符，它两个条件均不满足，则应当直接返回 $$\textit{false}$$；否则，当 $$\textit{typed}$$ 扫描完毕后，我们再检查 $$\textit{name}$$的每个字符是否都被「匹配」了。

实现上，我们使用两个下标 i,ji,j 追踪$$\textit{name}$$ 和 $$\textit{typed}$$ 的位置。

当 $$\textit{name}[i]=\textit{typed}[j]$$ 时，说明两个字符串存在一对匹配的字符，此时将 i,j都加一。

否则，如果$$\textit{typed}[j]=\textit{typed}[j-1]$$，说明存在一次长按键入，此时只将 j加1。

最后，如果 $$i=\textit{name}.\textit{length}$$，说明$$ \textit{name}$$ 的每个字符都被「匹配」了。

Day4 19. 删除链表的倒数第N个节点

官方做法：快慢指针

两个问题：

  1. 为什么不直接返回头head，而是返回ListNode *ans = dummy->next;

    当只有一个元素时，我们会删除该元素，此时head仍指向之前的内存位置，这样就会产生问题。而由于这种情况下second其实没有移动，为dummy,而second->next修改为了nullptr，也就是dummy->next修改为了nullptr，因此不会引起问题。

  2. 为什么需要一个头部节点，而不是first和second同时指向head

    一个出于上述原因， 使处理归一化。

    第二个原因是为了使second比first慢n+1个（first最终会走向最后一个元素的下一个），second如果不处理则会指向要删除的元素。这样给删除带来不便。
